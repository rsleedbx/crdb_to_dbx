# Contributing to CockroachDB to Databricks CDC Connector

We welcome contributions. Please follow these guidelines.

## How to contribute

1. **Open an issue** – For bugs, feature requests, or questions, open a [GitHub Issue](https://github.com/databricks/crdb_to_dbx/issues) (or the issue tracker for the repo you are contributing to).
2. **Pull requests** – Code changes should be submitted via **pull request**. The PR author and the reviewer should be different people. All content is peer-reviewed by at least one other team member or an SME.
3. **Scope** – Keep changes focused. For large changes, discuss in an issue first.

## Who to contact

- **Repo owners** – See the repository’s primary and secondary owners for access or permissions.
- **Security** – Report vulnerabilities per [SECURITY.md](SECURITY.md); do not open a public issue for security-sensitive findings.

## Guidelines

- Do not include customer data, PII, or proprietary information.
- Do not commit access tokens, PATs, or passwords (use placeholders or config examples only).
- Third-party code or assets must be acknowledged with their license (Apache, BSD, MIT, or Databricks license).

Thank you for contributing.
